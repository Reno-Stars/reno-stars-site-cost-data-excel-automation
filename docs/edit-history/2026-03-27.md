# 2026-03-27 — Refactor Excel Processor for Date-Range Sheet Format

## Summary

Major rewrite of `lib/excel-processor.ts` to support a new output Excel format where sheets are organized by date ranges (e.g., "1-3 26" = Jan-Mar 2026) containing multiple project blocks, replacing the previous format where each project had its own sheet tab.

## Files Modified

### `lib/excel-processor.ts` — Core logic rewrite

**New types added:**
- `ProjectBlock` — tracks data range, summary row positions, col C project names, and col A values per block
- `DateRange` — parsed date-range sheet metadata (startMonth, endMonth, year)

**New functions added:**
- `parseDateRange(name)` — parses sheet names like "1-3 26" into structured date ranges
- `getDateRangeSheets(workbook)` — filters workbook to date-range sheets, sorted most recent first
- `findProjectBlocks(sheet)` — scans for "单号" header rows, builds ProjectBlock array with summary rows and project identifiers
- `matchesAddressCode(text, addressCode)` — word-boundary regex match to prevent false positives (e.g., "53" won't match "1530")
- `findMatchingBlock(blocks, addressCode)` — searches blocks by col C then col A with word-boundary matching
- `isRowEmpty(sheet, r)` — checks if C/D/E/F/G/K columns are all empty
- `writeEntryToBlock(sheet, block, allBlocks, worker, entry, dateLabel)` — writes worker entry into matched project block

**Functions removed:**
- `extractAddressCode()` — was for old per-project sheet name matching
- `findSummaryRows()` — replaced by per-block tracking in `findProjectBlocks()`

**Functions replaced:**
- `writeEntryToSheet()` → `writeEntryToBlock()` — now operates within block boundaries, skips payment rows, preserves shared formulas, shifts subsequent blocks on row insertion

**`processFiles()` rewritten:**
- Iterates date-range sheets most recent first (instead of building address-to-sheet map)
- Pre-computes project blocks per sheet
- For each worker entry, searches all sheets for matching block

## Code Review Fixes Applied

After initial implementation, a code review identified and fixed:

1. **Critical: Cross-block row corruption** — `spliceRows` in one block now shifts row indices for all subsequent blocks on the same sheet via the `allBlocks` parameter
2. **Critical: False-positive matching** — Replaced `String.includes()` with word-boundary regex (`\b`) to prevent short codes like "53" from matching "1530"
3. **Medium: Search window** — Replaced magic 60-row constant with dynamic search bounded by next header row
4. **Medium: Empty row check** — Added column C to `isRowEmpty()` to prevent overwriting partially-filled rows

## Architecture Change

**Before:** One sheet per project, sheet names prefixed with address codes (e.g., "7217 Bridlewood"). Direct map lookup: `sheetMap.get(addressCode)`.

**After:** Sheets organized by date ranges, each containing ~11 project blocks. Matching requires scanning blocks within sheets in reverse chronological order, using word-boundary regex on col C/A values.

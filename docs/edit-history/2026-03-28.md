## Bug Fixes

### Material cost gap (user-reported)

**Symptom:** In the 5880 Dover block, material costs appeared at row 63 with empty K cells at rows 60–62, creating a visual gap.

**Root cause:** `writeMaterialToBlock` was called with `insertRow` (the worker's data row) as the search start, so materials were written next to their worker entry rather than packed from the top of the block.

**Fix:** Changed the `writeMaterialToBlock` call in `writeEntryToBlock` to search from `block.dataStartRow`, packing all material costs contiguously from the top of the block regardless of which worker row they belong to.

## Bug Fixes (continued)

### Shared formula error (user-reported)

**Symptom:** "Shared Formula master must exist above and or left of clone for cell H29" when writing output.

**Root cause:** ExcelJS shared formulas in col H use a master/clone model. `spliceRows` shifts rows but doesn't update shared formula references, leaving orphaned clones.

**Fix:** Added `breakSharedFormulasInColH(sheet)` — converts all shared formula clones to explicit `=Fn*Gn` formulas and strips shared master metadata BEFORE any writes/insertions.

### Row insertion splitting existing data (user-reported)

**Symptom:** Inserted worker rows appeared between existing data rows, splitting them apart.

**Root cause:** `findOrCreateInsertRow` inserted after the "last worker row" (col E filled), but existing data rows sometimes only have project info in C/D without a worker name in E.

**Fix:** Changed to insert before first payment row (or 总开销) instead of after last worker row.

### Buffer rows not being used (user-reported)

**Symptom:** New rows were being inserted before payments even though empty buffer rows existed after payment rows (between 第三笔款 and 总开销).

**Root cause:** `findOrCreateInsertRow` only searched for empty rows BEFORE payment rows. Buffer rows after payments were ignored.

**Fix:** Changed `findOrCreateInsertRow` to search the ENTIRE block for empty rows, skipping payment rows. Only insert before 总开销 if truly no empty rows exist anywhere in the block. Updated smoke test expectations: buffer rows at rows 8-27 now used for Block 1 (no insertion, no shift to later blocks).

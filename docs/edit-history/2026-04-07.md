# 2026-04-07 — Block-full insertion for materials, report improvements, gas column split

## Summary

Three categories of changes: (1) Fixed materials being silently dropped when a block's K column was full — now inserts a new row, same as worker entries. (2) Report improvements — worker/site totals now exclude unmatched entries, gas is a separate column from ticket, and input validation warnings surface missing fields. (3) Code review cleanup — removed dead `droppedMaterials` field, deduplicated `hasData` check, filtered zero-total workers from report.

## Bug Fixes

### Materials silently dropped when block K column full (user-reported)

**Symptom:** When all K cells in a block were occupied, material costs were silently dropped and counted as `droppedMaterials` instead of being written.

**Root cause:** `writeMaterialToBlock` returned `false` when no empty K cell existed, and callers just incremented a counter. Both materials-only entries and the material portion of regular entries were affected.

**Fix:** `writeMaterialToBlock` now calls `spliceBlockRow` to insert a new row before 总开销 when no empty K cell is found. Signature changed from `(sheet, searchFromRow, totalExpenseRow, ...)` returning `boolean` to `(sheet, block, allBlocks, ...)` returning `void`. Same insertion mechanism already used by `findOrCreateInsertRow` for worker entries.

## Feature Changes

### Report: worker/site totals exclude unmatched entries

**Before:** `computeCostSummaries` included ALL entries for worker totals, including those with unmatched addresses and all-zero entries.

**After:** Worker totals skip entries where `unmatchedAddresses.has(e.address)` or all four fields are zero. Workers with zero matched total are filtered out entirely via `.filter((wt) => wt.total > 0)`.

### Report: gas split from ticket as separate column

**Before:** `CostSummary.other` combined gas + ticket. UI showed a single "Other" column.

**After:** `CostSummary` has new `gas: number` field. `other` now contains only ticket. UI shows separate "Gas" and "Ticket" columns in all three tables (Extracted Worker Data, Employee Totals, Site Totals).

### Report: input validation warnings

**Before:** Rows with data but no address (D column) or no worker context (B/C columns) were silently skipped during parsing.

**After:** `parseInputFile` returns `{ workers, warnings }`. Three warning cases:
1. Worker row (B+C filled) has data in E/G/H/I but no address in D
2. Continuation row under a worker has data but no address in D
3. Row has address in D but no current worker context (before first worker or after Total)

Warnings displayed as red alert box in the UI with Chinese-language messages.

### Removed `droppedMaterials` from `ProcessResult`

Dead field — always 0 since materials now insert rows instead of being dropped. Removed from interface, callers, and UI (the orange warning banner was replaced by the new warnings system).
